# Wobbler — Max for Live / RNBO

## Prerequisites
- Max 9 (bundled with Ableton Live 11.3.1+)
- **RNBO Synth Building Blocks** package — install from Max Package Manager  
  `File > Show Package Manager > search "RNBO Synth Building Blocks"`

---

## Project Structure

```
max-for-live/
  wobbler-voice.rnbo        ← RNBO voice DSP (edit this)
  Wobbler-Dev.maxpat        ← Max test wrapper (use for development)
  Wobbler.amxd              ← M4L device (create when ready to deploy)
  exports/
    wobbler-voice.mxe64     ← generated by RNBO export (do not edit)
  README.md
```

## Development Workflow

### Testing in Max (no Live needed)

1. Open `Wobbler-Dev.maxpat` in Max 9
2. Enable DSP — click the **power button** in the Max toolbar
3. Play notes using the **kslider** in the patch, or connect a MIDI keyboard
4. Double-click the `rnbo~` object to open `wobbler-voice.rnbo` inside

### MIDI routing for kslider
The kslider → makenote → noteout approach needs a virtual MIDI loopback:
- **macOS**: Enable IAC Driver in Audio MIDI Setup (`Cmd+Space > Audio MIDI Setup > Window > Show MIDI Studio > IAC Driver > enable`)
- Or patch `kslider` directly into `midiin` via a Max `midiparse`/`midiformat` combo

### Editing the RNBO voice
- Double-click the `rnbo~` object in the dev wrapper to open `wobbler-voice.rnbo`
- Changes compile and take effect live
- Use `scope~` (already wired in dev wrapper) to inspect the output signal

## Phase 1 — Current State

**Signal flow**: `notein → mtof~ → cycle~ → *~ → out~ 1/2`  
**Envelope**: `velocity → sig~ → >~ 0. → adsr~ → *~` (amplitude)  
**Voice mgmt**: `adsr~ → thispoly~`  
**Params**: `attack` (ms), `decay` (ms), `sustain` (0–1), `release` (ms)

### Known items to verify in Max
- `adsr~` inlet config: if ADSR params don't work, try sending them as messages  
  `attack 100` to the message inlet of rnbo~ rather than direct param connections
- `@polyphony 8` voice stealing: RNBO handles automatically (oldest voice stolen)
- Param addressing from outside: `attack 100` (top-level params, no prefix needed when not in named subpatcher)

## Phase 2 — Next Steps
1. Replace `cycle~` with `saw~` and add `[param waveform @enum sine saw square triangle pulse]` + `selector~`
2. Add `svf~` filter between oscillator and envelope multiply:  
   `saw~ → svf~ → *~ (env)`
3. Add filter params: `filterCutoff`, `filterQ`, `filterType`
4. Add filter envelope amount: `adsr~ output × filterEnvAmt → add to cutoff`
5. Add velocity scaling: `latch~ + /~ 127` on the velocity path

## Deploying as Max for Live Device

When Phase 1 is working:

1. **Export Max External**: open `wobbler-voice.rnbo` → click export sidebar → select "Max External" → enable MIDI, Link Polyphony, voices=8 → Export
2. Create `Wobbler.amxd`:
   - New Max patcher → Save as `.amxd` in `~/Music/Ableton/User Library/Presets/Instruments/Max Instrument/`
   - Add `plugin~`, `live.thisdevice`, `midiin`, the exported external object, `live.gain~`, `plugout~`
   - Set device type to "MIDI Instrument" in device properties
3. **Freeze** the device to bundle dependencies: toolbar → Freeze Device button

---

## BPM Sync (Phase 4)
Inside `wobbler-voice.rnbo`, use `[phasor~ @lock 1]` — automatically syncs to Live's transport. No external BPM param needed.

## Export Targets (Phase 8)
From RNBO Export Sidebar:
- **Max External** → `exports/wobbler-voice.mxe64` (primary M4L target)
- **VST3 / AU** → standalone plugin (cloud compiled, no local SDK needed)
- **Web Export** → WASM/JS (replaces the existing Web Audio prototype)
- **C++ Source** → can replace `src/dsp/` in JUCE project
